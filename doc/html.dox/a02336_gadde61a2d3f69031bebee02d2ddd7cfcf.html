subroutine mfivop ( integer*8  fid,
character *(*)  name,
integer  access,
integer  major,
integer  minor,
integer  rel,
integer  cret 
)

Ouverture d'un fichier MED en indiquant la version du modèle à utiliser en cas de création d'un nouveau fichier.

Paramètres
fidIdentificateur du fichier.
nameNom du fichier.
accessMode d'acces au fichier.
majorNuméro de version majeur.
minorNuméro de version mineur.
relNuméro de release.
cretretour négatif en cas d'erreur (MED_ERR_HOWTO), Zéro sinon.
Voir également
MEDfileVersionOpen

Ouverture d'un fichier MED.

Remarques
  • La bibliothèque med-4.y1.z1/hdf5-1.10.(>2) accepte la création de fichier med 3.y.0/hdf5-1.8.6 sans la présence de la bibliothèque hdf5-1.8 (magie)
  • Le numéro minor ne peut être que dans la plage des versions mineures des bibliothèques distribuées avec le numéro majeur demandé.
  • Le numéro release n'est pas pris en compte. Le numéro release utilisé sera 0 (sans conséquence sur le modèle interne utilisé puisqu'il ne dépend pas du numéro de release).
  • Si aucun fichier de nom filename n'existe, il est crée dans les modes MED_ACC_RDWR et MED_ACC_RDEXT.
  • En mode MED_ACC_CREAT un nouveau fichier filename est crée qu'il en existe déjà un ou non.

Définition à la ligne 20 du fichier medfile.f.
